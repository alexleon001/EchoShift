# EchoShift

A rhythm pattern puzzle game for iOS and Android. The grid lights up in sequence — replicate the pattern before it gets harder. Every level is dynamically generated by Claude API, ensuring infinite variety and adaptive difficulty.

**Tagline:** *"Infinite. Addictive. Never the same twice."*

## Quick Start (5 steps)

### 1. Clone and install

```bash
git clone <repo-url>
cd EchoShift
npm install
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env with your API keys (Supabase, Anthropic, etc.)
```

### 3. Install server dependencies

```bash
cd server
npm install
cd ..
```

### 4. Start the backend

```bash
npm run server:dev
# Server runs on http://localhost:3000
```

### 5. Start the app

```bash
npx expo start
# Scan QR with Expo Go, or press 'i' for iOS / 'a' for Android
```

> **Node 24 users:** If you see `ERR_UNSUPPORTED_NODE_MODULES_TYPE_STRIPPING`, run with:
> ```bash
> NODE_OPTIONS="--no-experimental-strip-types" npx expo start
> ```

## Game Modes

| Mode | Description |
|------|-------------|
| **Endless** | No limits — pattern grows indefinitely |
| **Blitz** | 60 seconds — max points under pressure |
| **Daily** | Same pattern for everyone, 3 attempts |
| **Duo** | 1v1 real-time (Phase 3) |

## Tech Stack

- **App:** React Native + Expo SDK 52, TypeScript strict
- **Rendering:** React Native Skia, Reanimated 3
- **State:** Zustand, TanStack Query
- **Backend:** Node.js 22 + Fastify 5
- **AI:** Claude API (Anthropic SDK) for pattern generation
- **Database:** Supabase (PostgreSQL + Auth)
- **Cache:** Redis (Upstash) for pattern caching
- **Monetization:** RevenueCat (IAP), Google AdMob (rewarded)

## Environment Variables

All variables are documented in `.env.example`. Key services:

| Variable | Required | Description |
|----------|----------|-------------|
| `ANTHROPIC_API_KEY` | Server | Claude API for pattern generation |
| `EXPO_PUBLIC_SUPABASE_URL` | Both | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_ANON_KEY` | Client | Supabase anonymous key |
| `SUPABASE_SERVICE_KEY` | Server | Supabase service role key |

## Architecture

```
Client (React Native)  ──HTTPS──>  Server (Fastify)  ──>  Claude API
       │                                │
       └── Zustand Store                └── Redis Cache
           │                                │
           └── MMKV (offline)               └── Supabase (PostgreSQL)
```

The client never calls Claude API directly. All AI generation goes through the server, which validates responses with Zod and caches patterns in Redis.
